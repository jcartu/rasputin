## 🔐 Security Notes

**Public URLs:**
- Free Cloudflare tunnels (no authentication)
- Anyone with URL can access
- Production deployment should use:
  - Named Cloudflare tunnels with auth
  - Or deploy to Vercel/Netlify with proper auth
  - Environment variables for sensitive data

**Credentials:**
- OpenClaw Gateway: Local access only
- Qdrant: Local access only
- Database: Local access only
- Public URLs expose frontend/backend API only

## 🚀 Production Deployment (Recommended)

### **Frontend → Vercel**
```bash
cd ~/.openclaw/workspace/alfie-ui
vercel deploy --prod
```

### **Backend → Railway/Render/Fly.io**
```bash
cd ~/.openclaw/workspace/alfie-backend
# Configure env vars first
railway up  # or render deploy, or fly deploy
```

### **Named Cloudflare Tunnel**
```bash
cloudflared tunnel login
cloudflared tunnel create alfie
cloudflared tunnel route dns alfie alfie.yourdomain.com
cloudflared tunnel run alfie
```
